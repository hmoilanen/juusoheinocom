-editable-content:
  -muuta komponenttia niin että kun sisältöä klikataan, aukeaa "modal", jossa editointi tapahtuu
  -tee tarkastelu, onko kyseessä kuva
  -jos kuva, tallenna kuva storageen
  -tee toiminnallisuus, jossa "modaalissa" voi editoida
    -...sekä yksittäistä
    -...että montaa asiaa kerralla

  -toimintaperiaate:
    -wrappaa mikä tahansa komponentti editable-content:tiin

  -modaalissa:
    -tarkastele mitä dataa tulee sisään
    -jos yksittäinen -> string
      -tarkastele joka kerta regexillä onko string kuva!
    -jos objekti (älä hyväksy arrayta)...
      -tarkastele jokainen objektin stringi (älä hyväksy muuta sisältöä objektissa!)



-asioita joita modaallissa tarvitaan / tarekastellaan
  -path
  -miitä inputtityyppiä käytetään
    -base-input
    -base-textarea (onko sittenki default? eikä input)
    -dropzone
    --base-radio ? (tämän voi lisätä joskus myöhemminkin -> ei tue mieleen järkevää käyttötapausta)
    --listat / arrayt (voi myös lisätä myöhemmin -> ...tai muuttaa suoraan kantaan)

-modaali voisi ottaa vastaan datan näin:
  -[] tai {}
    -jos {} = perustapaus -> yksi muutettava tietue
    -jos [...{}] = monta muutettavaa tietuetta
      -sisältö: { path: xxx, type: input/textarea/dropzone }
        -> tässä se hyvä puolii että näitä voi nyt tehdä itse sellaisella kokoonpanolla kuin milloinkin tarvii

  -text/home/refs/someId
